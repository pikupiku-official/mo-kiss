# ルビ・傍点 実装設計

## 概要

ドキドキスクリプト Ver.2 にルビ（振り仮名）と傍点を追加する。
既存の1文字グリッド描画を拡張する形で実装する。

---

## 構文

```
{ベーステキスト|よみ}   ; ルビ
{boten:対象テキスト}    ; 傍点
```

---

## 実装フェーズ

### Phase 1: テキストパーサー拡張

**対象ファイル：** `dialogue/text_renderer.py`（または共通ユーティリティに切り出し）

#### パース対象

| 記法 | 種別 | 区別方法 |
|------|------|---------|
| `{苗字}` | 変数置換 | `|` なし、`boten:` なし |
| `{愛沼\|あいぬま}` | ルビ | `|` あり |
| `{boten:絶対に}` | 傍点 | `boten:` プレフィックス |

#### パース結果の内部表現

文字列を「トークンリスト」に変換する。

```python
# 型定義
@dataclass
class PlainChar:
    char: str

@dataclass
class RubySpan:
    base: str    # ベーステキスト（複数文字可）
    ruby: str    # よみ（複数文字可）

@dataclass
class BotenSpan:
    base: str    # 傍点を振るテキスト

Token = PlainChar | RubySpan | BotenSpan
```

#### パース関数

```python
def parse_inline_markup(text: str) -> list[Token]:
    """
    変数置換済みテキストをトークンリストに変換する。
    name_manager.substitute_variables() の後に呼ぶこと。
    """
    tokens = []
    pattern = re.compile(r'\{([^}|]+)\|([^}]+)\}|\{boten:([^}]+)\}')
    last = 0
    for m in pattern.finditer(text):
        # マッチ前の平文
        for ch in text[last:m.start()]:
            tokens.append(PlainChar(ch))
        if m.group(1) is not None:
            # ルビ
            tokens.append(RubySpan(base=m.group(1), ruby=m.group(2)))
        else:
            # 傍点
            tokens.append(BotenSpan(base=m.group(3)))
        last = m.end()
    for ch in text[last:]:
        tokens.append(PlainChar(ch))
    return tokens
```

#### 折り返し計算

```python
def count_chars_for_wrap(tokens: list[Token]) -> int:
    """26文字折り返し用のカウント（ベース文字数のみ）"""
    n = 0
    for t in tokens:
        if isinstance(t, PlainChar):
            n += 1
        else:
            n += len(t.base)  # RubySpan / BotenSpan ともベース文字数
    return n
```

### Phase 2: グリッド描画拡張

**対象ファイル：** `dialogue/text_renderer.py`

#### 行高の変更

ルビがある行は行高を増やす必要がある。

```python
RUBY_FONT_RATIO = 0.45   # ルビフォントサイズ = 本文フォント × 0.45
RUBY_MARGIN_PX  = 2      # ルビとベース文字の隙間（px）

# 行全体の高さ
base_h    = font.get_height()
ruby_h    = int(base_h * RUBY_FONT_RATIO) + RUBY_MARGIN_PX
line_h    = base_h + ruby_h   # ルビ領域 + 本文領域
```

#### 描画ロジック（PlainChar）

```
[ruby_area: 空白]
[base_area: 文字]
```

```python
# Y オフセット（ルビ領域分だけ下げる）
base_y = ruby_h
char_surf = render_char(char)
line_surf.blit(char_surf, (grid_x, base_y))
```

#### 描画ロジック（RubySpan）

```
[あいぬま] ← ルビ（小フォント、均等割り付け）
[愛　　沼] ← ベース（通常フォント）
```

```python
def render_ruby_span(span: RubySpan, grid_x: int, grid_char_width: int) -> Surface:
    base_chars = len(span.base)
    span_width = grid_char_width * base_chars

    surf = Surface((span_width, line_h), SRCALPHA)

    # ベース文字を描画
    for i, ch in enumerate(span.base):
        ch_surf = render_char(ch)
        surf.blit(ch_surf, (i * grid_char_width, ruby_h))

    # ルビを均等割り付けで描画
    ruby_surf = ruby_font.render(span.ruby, True, color)
    ruby_x = (span_width - ruby_surf.get_width()) // 2  # 中央寄せ
    surf.blit(ruby_surf, (ruby_x, 0))

    return surf
```

> **Note:** ルビ文字列がベース幅より広い場合（例：1文字ベースに長いよみ）は、
> ルビをスパン幅にスケールするか、はみ出しを許容するか後で決める。

#### 描画ロジック（BotenSpan）

```
[・　・　・] ← 傍点（ルビと同じ小フォント領域）
[絶　対　に] ← ベース
```

```python
def render_boten_span(span: BotenSpan, grid_x: int, grid_char_width: int) -> Surface:
    base_chars = len(span.base)
    span_width = grid_char_width * base_chars

    surf = Surface((span_width, line_h), SRCALPHA)

    for i, ch in enumerate(span.base):
        # ベース文字
        ch_surf = render_char(ch)
        surf.blit(ch_surf, (i * grid_char_width, ruby_h))

        # 傍点（各文字の中央上）
        dot_surf = ruby_font.render("·", True, color)
        dot_x = i * grid_char_width + (grid_char_width - dot_surf.get_width()) // 2
        surf.blit(dot_surf, (dot_x, 0))

    return surf
```

### Phase 3: バックログ対応

**対象ファイル：** `dialogue/backlog_manager.py`

バックログも同じトークンリスト → 同じ描画ロジックを使う。

- `parse_inline_markup()` をバックログの行描画前に呼ぶ
- `render_ruby_span()` / `render_boten_span()` を共通モジュールに切り出して両者で使う

**共通モジュール案：** `dialogue/inline_markup.py`

```
dialogue/
  inline_markup.py   ← parse_inline_markup(), RubySpan, BotenSpan, PlainChar
  text_renderer.py   ← import inline_markup
  backlog_manager.py ← import inline_markup
```
