# データとアセット設計（To‑Be）

本書は **データの区分・保存場所・命名方針** を整理し、  
アセット運用とセーブ運用の“軸”を定義する。

---

## 1. データ区分（概念）
### 1) セーブ状態
- ゲーム進行の復元に必要な状態  
- 例: 現在のBase State / 時間 / 進行度 / 直前イベント

### 2) シナリオフラグ
- イベント完了・分岐選択など  
- 原則として **DIALOGUEで更新**

### 3) 時間/日付
- イベント終了で進める  
- MAPでイベントが無い場合の時間スキップあり（予定）

### 4) プレイヤー名
- MAIN_MENUの「はじめから」で決定  
- セーブスロット内で固定

### 5) 設定（OPTION）
- 音量 / キー / 表示 / 速度 / スキップ など  
- **即時反映・永続化**

---

## 2. 保存場所（現状を前提にする）
### current_state（現状の保持）
- `data/current_state/`
  - `completed_events.csv`
  - `time_state.json`
  - `player_name.json`

### templates（初期化）
- `data/templates/`
  - `completed_events_template.csv`
  - `time_state_template.json`
  - `player_name_template.json`

### save（セーブスロット）
- `data/save/`
  - `saveslot_01/` など
  - 複数スロットを前提

### events（イベント資産）
- `events/`
  - `events.csv`
  - `story_flags.json`
  - `E001.ks` 〜 `E066.ks`

---

## 3. セーブ方針
- **スロット単位で別世界線**
- 1スロット = 1つの「はじめから」  
  → プレイヤー名は固定  
- ロードは **保存時の状態へ復帰**
