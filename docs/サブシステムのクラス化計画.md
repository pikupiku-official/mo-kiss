# サブシステムのクラス化計画

## 目的

現在、サブシステム間の遷移時に音声（BGM・SE）を停止する処理が、各`switch_to_*`メソッドに重複して記述されている。この設計は保守性が低く、新しいサブシステムを追加する際に組み合わせ爆発を起こす。

本計画では、全サブシステムに統一されたインターフェースを持たせることで、この問題を解決する。

---

## ⚠️ 再検査で判明した追加問題（2026-05-10）

計画書初版（2025-11-23）に対して実コードとの乖離を再検査した結果、以下の見落としが判明した。
フェーズごとの実施前に必ず確認すること。

### 追加問題A（🔴重大）— dialogue が独自に `pygame.event.get()` を呼ぶ

`dialogue/controller2.py:135`
```python
def handle_events(game_state, screen):
    for event in pygame.event.get():   # ← 自分でget()している
```

フェーズ4の統一メインループ案（`result = self.current_subsystem.handle_events(events)`）は
**dialogue に対しては機能しない**。

**対処方針（フェーズ3に明記）:**
`DialogueSubsystem.handle_events(events)` は引数 `events` を無視し、
内部で `controller2.handle_events(game_state, screen)` を呼ぶ。
インターフェース上は統一されるが、dialogue だけは events を受け取らない特殊実装になる。
将来的に controller2 を改修して events 引数を受け取るようにすれば完全統一できる。

### 追加問題B（🔴重大）— 座標系切り替えが cleanup/on_enter に未反映

`switch_to_dialogue` では `config.OFFSET_X/Y/SCALE` を 0/1.0 にリセットし、
`switch_to_map`/`switch_to_menu` で復元している。

**対処方針（フェーズ3に追記）:**
- `DialogueSubsystem.on_enter()` で座標系をゼロリセット（＆元の値を自身に保存）
- `DialogueSubsystem.cleanup()` で座標系を復元

### 追加問題C（🟡中）— `AdvancedKimikissMap.__init__` が `screen` を引数に取らない

```python
class AdvancedKimikissMap:
    def __init__(self):                          # screenなし
        self.screen = pygame.display.get_surface()
```

`SubsystemBase.__init__(self, screen)` を継承するには Map 側のシグネチャ変更が必要。

**対処方針（フェーズ2に追記）:**
`def __init__(self, screen=None):` に変更し、
`None` の場合は `pygame.display.get_surface()` にフォールバックする。

### 追加問題D（🟡中）— `map.py` 内に dialogue 実行コードが残存

`launch_main_game`（L1358）、`run_ks_event_in_window`（L1592）など
**約600行の旧設計 dialogue 実行コードが map.py に残っている**。
フェーズ2完了後もこの残骸が残り、将来のバグ温床になる。

**対処方針（フェーズ2完了後の追加タスク）:**
これらメソッドが main.py 経由の switch_to_dialogue() で代替済みであることを確認後、
別ブランチで削除する（フェーズ2とは分離してリスク管理すること）。

### 追加問題E（🟢軽微）— `MainMenu.handle_event(event)` と統一 I/F の不一致

MainMenu は単一イベントを受け取る `handle_event(event)` を持つ。
統一 I/F は `handle_events(events: list)` を想定。
**対処方針:** フェーズ1 で `handle_events(events)` ラッパーを追加する。

## 現在の問題点

### 問題1: 組み合わせ爆発

```python
# 現状: 各switch_to_*に全ての遷移元をチェック
def switch_to_menu():
    if current_mode == "dialogue": dialogue停止処理
    if current_mode == "map": map停止処理
    # 将来homeやtitleが音を持ったら...さらに増える

def switch_to_home():
    if current_mode == "dialogue": dialogue停止処理
    if current_mode == "map": map停止処理
    # ...同じ処理を繰り返し
```

**問題点:**
- サブシステムがN個あると、N×Nの組み合わせを書く必要がある
- コードの重複が多い
- 新しいサブシステム追加時の修正箇所が多い
- バグが混入しやすい

### 問題2: 責任の所在が不明確

各サブシステムが自分の終了処理を知らず、main.pyが全て把握している状態。

### 問題3: 統一されたライフサイクルがない

- 初期化: バラバラ（クラス、関数、辞書...）
- イベント処理: `handle_events`, `handle_event` など名前が統一されていない
- 更新: `update` はあるが、使い方が統一されていない
- 描画: `render` はあるが、呼び出し方がバラバラ
- 終了: cleanup処理が存在しない

## 目標アーキテクチャ

### 統一された基底クラス

```python
# subsystem_base.py（新規ファイル）
from abc import ABC, abstractmethod
import pygame

class SubsystemBase(ABC):
    """全サブシステムの基底クラス"""

    def __init__(self, screen: pygame.Surface):
        self.screen = screen

    @abstractmethod
    def handle_events(self, events) -> str | None:
        """
        イベント処理

        Returns:
            str | None: 遷移先（例: "go_to_map", "go_to_menu"）
                       Noneの場合は遷移しない
        """
        pass

    @abstractmethod
    def update(self):
        """ゲームロジック更新"""
        pass

    @abstractmethod
    def render(self):
        """画面描画"""
        pass

    def cleanup(self):
        """
        サブシステム終了時の処理（オーバーライド可能）
        デフォルトは何もしない
        """
        pass

    def on_enter(self):
        """
        サブシステムに入った時の処理（オーバーライド可能）
        例：BGM再生開始、初期化など
        """
        pass
```

### main.pyでの統一された切り替え処理

```python
class GameApplication:
    def switch_to(self, subsystem: SubsystemBase, mode_name: str):
        """サブシステム切り替えの統一メソッド"""

        # 1. 現在のサブシステムをクリーンアップ
        if self.current_subsystem:
            self.current_subsystem.cleanup()
            print(f"🔇 {self.current_mode} cleanup完了")

        # 2. 新しいサブシステムに切り替え
        self.current_subsystem = subsystem
        self.current_mode = mode_name

        # 3. 新しいサブシステムの初期化処理
        self.current_subsystem.on_enter()
        print(f"✅ {mode_name} に切り替え")
```

### フェーズ3: dialogueシステムの移行（優先度：中）

dialogueは現在**辞書形式のgame_state**なので、2つのアプローチがある。

#### アプローチA: ラッパークラスを作る（推奨：段階的移行）

**実施内容:**
1. `dialogue/dialogue_subsystem.py` を新規作成
2. `DialogueSubsystem` クラスを実装（ラッパー）
3. 既存のgame_stateを内部で保持
4. controller2、model、scenario_managerは既存のまま使用

> ⚠️ **追加問題A・B 対応（必須）:**
> - `handle_events(events)` の `events` 引数は **無視**し、
>   内部で `controller2.handle_events(self.game_state, self.screen)` を呼ぶ
> - `on_enter()` で `config.OFFSET_X/Y = 0`, `config.SCALE = 1.0` にリセット
>   （元の値は `self._saved_offset_x` 等に退避）
> - `cleanup()` で退避した座標系を復元

**変更例:**
```python
# dialogue/dialogue_subsystem.py（新規ファイル）
from subsystem_base import SubsystemBase
from dialogue.model import initialize_game as init_dialogue_game
from dialogue.controller2 import handle_events as dialogue_handle_events
from dialogue.controller2 import update_game as dialogue_update

class DialogueSubsystem(SubsystemBase):
    """dialogueシステムのラッパー"""

    def __init__(self, screen, event_file=None):
        super().__init__(screen)

        # 既存のgame_state辞書を内部で保持
        self.game_state = init_dialogue_game()

        # イベントファイルの読み込み
        if event_file:
            self._load_event_file(event_file)

    def _load_event_file(self, event_file):
        """イベントファイル読み込み"""
        from dialogue.dialogue_loader import DialogueLoader
        from dialogue.data_normalizer import normalize_dialogue_data

        dialogue_loader = DialogueLoader()
        raw_dialogue_data = dialogue_loader.load_dialogue_from_ks(event_file)
        if raw_dialogue_data:
            dialogue_data = normalize_dialogue_data(raw_dialogue_data)
            if dialogue_data:
                self.game_state['dialogue_data'] = dialogue_data
                self.game_state['current_paragraph'] = -1
                from dialogue.model import advance_dialogue
                advance_dialogue(self.game_state)

    def handle_events(self, events) -> str | None:
        """dialogueのイベント処理"""
        continue_dialogue = dialogue_handle_events(self.game_state, self.screen)

        if not continue_dialogue:
            return "dialogue_ended"

        return None

    def update(self):
        """dialogueの更新"""
        dialogue_update(self.game_state)

    def render(self):
        """dialogueの描画"""
        from dialogue.text_renderer import TextRenderer
        from dialogue.character_manager import draw_characters
        from dialogue.background_manager import draw_background
        from dialogue.choice_renderer import ChoiceRenderer
        from dialogue.fade_manager import draw_fade_overlay

        # 背景描画
        draw_background(self.game_state)

        # キャラクター描画
        draw_characters(self.game_state)

        # フェードオーバーレイ描画
        draw_fade_overlay(self.game_state)

        # UIエレメント描画
        # ... 既存の描画コード

    def cleanup(self):
        """dialogue終了時の処理"""
        self.game_state['bgm_manager'].stop_bgm()
        self.game_state['se_manager'].stop_all_se()
        print("🔇 dialogue cleanup: BGM・SE停止")
```

**メリット:**
- 既存のdialogueコードをほぼそのまま使える
- 段階的に移行できる
- リスクが低い

**デメリット:**
- ラッパーが増える
- 完全なリファクタリングではない

**完了条件:**
- DialogueSubsystemクラスが動作する
- 既存のdialogue機能が全て使える
- cleanup()でBGM・SEが停止する

**実施内容:**

**変更例:**
```python
# main.py
class GameApplication:
    def __init__(self):
        self.screen = None
        self.current_subsystem: SubsystemBase | None = None
        self.current_mode = "menu"

    def switch_to(self, subsystem: SubsystemBase, mode_name: str):
        """サブシステム切り替えの統一メソッド"""

        # 1. 現在のサブシステムをクリーンアップ
        if self.current_subsystem:
            self.current_subsystem.cleanup()
            print(f"🔇 {self.current_mode} cleanup完了")

        # 2. 新しいサブシステムに切り替え
        self.current_subsystem = subsystem
        self.current_mode = mode_name

        # 3. 新しいサブシステムの初期化処理
        self.current_subsystem.on_enter()
        print(f"✅ {mode_name} に切り替え")

    def switch_to_menu(self):
        if not self.main_menu:
            self.main_menu = MainMenu(self.screen)
        self.switch_to(self.main_menu, "menu")

    def switch_to_map(self):
        if not self.map_system:
            self.map_system = AdvancedKimikissMap(self.screen)
        self.switch_to(self.map_system, "map")

    def switch_to_dialogue(self, event_file):
        dialogue = DialogueSubsystem(self.screen, event_file)
        self.switch_to(dialogue, "dialogue")

    def switch_to_home(self):
        if not self.home_module:
            self.home_module = HomeModule(self.screen)
        self.switch_to(self.home_module, "home")

    def run(self):
        """メインゲームループ"""
        while self.running:
            events = pygame.event.get()

            # イベント処理
            if self.current_subsystem:
                result = self.current_subsystem.handle_events(events)

                # 遷移処理
                if result:
                    self._handle_transition(result)

            # 更新
            if self.current_subsystem:
                self.current_subsystem.update()

            # 描画
            if self.current_subsystem:
                self.current_subsystem.render()

            pygame.display.flip()
            self.clock.tick(30)

    def _handle_transition(self, result: str):
        """遷移結果を処理"""
        if result == "go_to_map":
            self.switch_to_map()
        elif result == "go_to_menu":
            self.switch_to_menu()
        elif result == "go_to_home":
            self.switch_to_home()
        elif result.startswith("start_event:"):
            event_id = result.split(":")[1]
            self.switch_to_dialogue(f"events/{event_id}.ks")
        elif result == "dialogue_ended":
            self.mark_current_event_as_completed()
            self.switch_to_map()
        elif result == "quit":
            self.running = False
```

**削除できるコード:**
```python
# これらの重複した処理が全て不要になる

    self.dialogue_game_state['se_manager'].stop_all_se()

if self.current_mode == "map" and self.map_system:
    self.map_system.bgm_manager.stop_bgm()
```

**完了条件:**
- 全てのサブシステム切り替えがswitch_to()を経由
- 重複コードが削除されている
- 音声が適切に停止される
